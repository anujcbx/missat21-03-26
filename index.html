What is LAMBDA?

LAMBDA is a way to give your own name to any formula you keep writing again and again — so you never have to write it twice.

🧑‍🏫

Think of it this way: You know how =SUM() is a formula someone built and gave a name? LAMBDA lets YOU do the same thing. You write your own logic once, give it a name, and then just call that name anywhere in your sheet.

Without LAMBDA

You write this formula in every single row:

=IF(C2-B2>3,"Delayed","On Time")

Need to change "3" to "5"? You fix 500 cells.

With LAMBDA

You define it once, call it anywhere:

Define once in Data → Named Functions: =LAMBDA(dispatch, sla, IF(dispatch-sla>3,"Delayed","On Time")) Then call it in any cell: =DELAY_CHECK(C2, B2)

Change "3" once → all cells update instantly.

λ How To

How to Create a Named Function

It takes 2 minutes. Here's exactly what to click, step by step.

1

Click "Data" in the top menu

In Google Sheets, go to the top menu bar → click Data → scroll to the bottom → click Named Functions.

2

Click "+ Add new function"

A panel opens on the right side of your screen. This is where you'll type your formula.

3

Give it a name — all CAPS, no spaces

Examples: NET_SALARY, DELAY_FLAG, GST_TOTAL. This is what you'll type in cells later.

4

Add argument placeholders — these are your inputs

Click + Add argument placeholder and name each input. Example: for a salary formula, add basic. For a delay check, add dispatch_date and sla_days.

5

Write your formula using those argument names

Instead of writing B2, write basic. Instead of C2, write dispatch_date. The formula works exactly the same way.

6

Click Create — and it's ready!

Now type =NET_SALARY(B2) in any cell. It autocompletes just like =SUM or =IF. Your whole team can use it.

🧑‍🏫

One important thing: Named Functions are saved inside that specific Google Sheet. If you want to use the same functions in a new sheet, go to Data → Named Functions → click the three dots → Copy to clipboard. Then paste into the new sheet. This way you build a personal formula library over time.

λ MAP

MAP — Apply your formula to every row

MAP is the most useful LAMBDA helper. It takes your Named Function and applies it to an entire column in one shot — no dragging, no copying down.

🧑‍🏫

The simplest way to understand MAP: Imagine you have 500 employee names in column A and you want to add "Mr./Ms." before each one. Instead of writing a formula 500 times, MAP does it for all 500 in ONE cell. That's it.

MAP

MAP(range1, [range2], lambda_or_named_function)

Apply a formula to every item in a range. Returns results for each row.
Use MAP when: You want to run the same formula on every row of a column. It's the replacement for "dragging a formula down".
Example 1 — Add GST to every price in column B
Instead of writing =B2*1.18 in C2 and dragging down… =MAP(B2:B500, LAMBDA(price, price * 1.18)) Or if you saved it as a Named Function GST_PRICE: =MAP(B2:B500, GST_PRICE)
Example 2 — Check delay status for all 500 trucks (2 input columns)
B = dispatch date, C = SLA days =MAP(B2:B500, C2:C500, LAMBDA(dispatch, sla, IF(TODAY()-dispatch>sla, "🔴 Delayed", "✅ On Time")) )
Example 3 — Grade all students in one formula
C = marks, D = total marks =MAP(C2:C100, D2:D100, STUDENT_GRADE)
StudentMarksTotalGrade (auto via MAP)
Arjun88100A (88%)
Priya62100C (62%)
Rohit35100F — Fail
✅ When to use MAP vs ARRAYFORMULA? Use MAP when your logic is inside a Named Function. Use ARRAYFORMULA for simple inline math like =ARRAYFORMULA(B2:B500*1.18). Both work — MAP is cleaner for complex logic.
λ BYROW

BYROW — Do something for each row

BYROW runs your formula on each row of a range. Perfect when you need to do something across multiple columns within a single row.

🧑‍🏫

MAP vs BYROW — what's the difference? MAP works on individual cells (one column at a time). BYROW works on an entire row at once. So if you want to find the highest score across 5 subject columns for each student — that's BYROW territory.

BYROW

BYROW(range, LAMBDA(row, formula))

Runs your formula once per row — the whole row is passed in as one unit.
Use BYROW when: You want to do something across multiple columns in a row — like finding the total, max, min, or a custom check across subjects/months/products for each person.
Example 1 — Total sales across Jan, Feb, Mar for each rep (columns B, C, D)
=BYROW(B2:D10, LAMBDA(row, SUM(row)) )
Example 2 — Find the best month score for each student
Students in rows, months in columns B to F =BYROW(B2:F20, LAMBDA(row, MAX(row)) )
Example 3 — Flag if any column in the row is blank (data completeness check)
=BYROW(A2:E100, LAMBDA(row, IF(COUNTBLANK(row)>0, "⚠️ Incomplete", "✅ Complete")) )
EmployeeJanFebMarBest Month (BYROW)
Sneha85,00092,00078,00092,000
Kiran60,00055,00070,00070,000
Remember: BYROW gives you ONE result per row. So if you have 50 rows, you get 50 results — one per student, employee, or item.
λ BYCOL

BYCOL — Do something for each column

BYCOL is the column version of BYROW. It runs your formula once per column — giving you one result for each column in your range.

🧑‍🏫

When would you use this? Say you have 12 months as columns and 50 employees as rows. You want to know the average attendance for each month. That's BYCOL — one result per month column.

BYCOL

BYCOL(range, LAMBDA(col, formula))

Runs your formula once per column — gives one result for each column.
Use BYCOL when: Your data has multiple columns (months, products, subjects) and you want a summary for each column — average, max, count, etc.
Example 1 — Average score per subject (columns B to F = 5 subjects)
=BYCOL(B2:F50, LAMBDA(col, AVERAGE(col)) )
Example 2 — Count how many students passed in each subject (pass = score ≥ 40)
=BYCOL(B2:F50, LAMBDA(col, COUNTIF(col, ">="&40)) )
Example 3 — Find peak sales month across all products
Each column = one month, rows = products =BYCOL(B2:M20, LAMBDA(col, SUM(col)) )
Subject →MathsScienceEnglish
Average (BYCOL)72.461.868.5

Quick rule: BYROW → one result per ROW. BYCOL → one result per COLUMN. They're mirror images of each other.
λ REDUCE

REDUCE — Build a running total with your own rules

REDUCE goes through a list of values one by one, accumulating a result as it goes. It's like a rolling calculation that you design yourself.

🧑‍🏫

Imagine this: You're keeping a tally. You start at 0. For every sale above ₹10,000 you add 1. For every sale below, you add 0. REDUCE does that tally automatically across hundreds of rows. SUM can't do this with conditions — REDUCE can.

REDUCE

REDUCE(initial_value, range, LAMBDA(accumulator, current, formula))

Accumulates a single final result by processing each item. Gives you ONE final answer.
Use REDUCE when: You want one final number built up from many rows — like "how many orders were above target?" or "total bonus pool where only eligible employees are counted".
Example 1 — Count how many deliveries were on time (simple tally)
Column C has "On Time" or "Delayed" for each delivery =REDUCE(0, C2:C200, LAMBDA(acc, val, acc + IF(val="On Time", 1, 0)) ) → Returns: 142 (out of 200 deliveries)
Example 2 — Total bonus only for employees who hit 100% target
Column D = achievement %, Column E = salary =REDUCE(0, D2:D50, LAMBDA(acc, idx, acc + IF(INDEX(D2:D50, idx) >= 100, INDEX(E2:E50, idx) * 0.1, 0)) )
Example 3 — Simplest use: sum only positive numbers
=REDUCE(0, B2:B100, LAMBDA(acc, val, acc + IF(val>0, val, 0)) )
REDUCE vs SUM: SUM adds everything. REDUCE adds only what your rule says to add. Use REDUCE when SUMIF isn't powerful enough for your condition.
λ SCAN

SCAN — Show the running total at every step

SCAN is like REDUCE's twin — but instead of giving you just the final answer, it shows you the running result at every single row. Great for running balances and cumulative totals.

🧑‍🏫

The difference in one line: REDUCE says "what's the total at the end?" SCAN says "what's the running total after each row?" Think of your bank passbook — SCAN gives you that column of running balance.

SCAN

SCAN(initial_value, range, LAMBDA(accumulator, current, formula))

Same as REDUCE but returns a running result at every row — not just the final total.
Use SCAN when: You need a running balance, cumulative total, or progressive count — visible at every row. Perfect for cash flow sheets, stock ledgers, or monthly sales cumulative.
Example 1 — Running cash balance (receipts in column B, payments in column C)
=SCAN(0, B2:B100, LAMBDA(balance, receipt, balance + receipt - OFFSET(receipt, 0, 1)) )
Example 2 — Simpler: running total of sales, row by row
Column B = daily sales. Get cumulative sales column: =SCAN(0, B2:B30, LAMBDA(total, day_sale, total + day_sale) )

REDUCE returns 1 cell. SCAN returns a column. Use SCAN when you want to fill a whole column with cumulative values automatically.
λ MAKEARRAY

MAKEARRAY — Build a table automatically

MAKEARRAY creates a grid of values — you tell it how many rows and columns you want, and what formula to put in each cell. It fills the whole table for you.

🧑‍🏫

Think of it like this: You want to create a 5×5 multiplication table. Normally you'd write formulas in 25 cells. With MAKEARRAY you write ONE formula and it fills all 25 cells automatically. Same for any grid — a rate card, a comparison table, a paygrade matrix.

MKARY

MAKEARRAY(rows, cols, LAMBDA(row_num, col_num, formula))

Generates a full grid. You get the row number and column number for each cell to use in your formula.
Use MAKEARRAY when: You need to auto-build a structured table — rate cards, comparison grids, schedule matrices, multiplication tables, or any table where each cell = a formula of its row and column position.
Example 1 — Multiplication table (rows 1–5, columns 1–5)
=MAKEARRAY(5, 5, LAMBDA(r, c, r * c) )
Example 2 — Freight rate card (5 vehicle types × 6 distance zones)
Base rates in column A (vehicle types), distance multipliers in row 1 =MAKEARRAY(5, 6, LAMBDA(r, c, INDEX(A2:A6, r) * INDEX(B1:G1, c)) )
Example 3 — Auto-fill a 7-day shift schedule header
=MAKEARRAY(1, 7, LAMBDA(r, c, TEXT(TODAY()-WEEKDAY(TODAY())+c, "DDD DD-MMM")) )
Key point: MAKEARRAY gives you the row number (r) and column number (c) for each cell. Your formula uses r and c to decide what goes in that position.

NET_SALARY — Take-home salary

One argument: basic salary. Returns net pay after HRA, DA, PF and PT.

Argument: basic

ATTEND_STATUS — Attendance flag

present days ÷ total days → Regular / At Risk / Problem

Arguments: present, total
=LAMBDA(p,t,LET(pct,p/t*100,IFS(pct>=90,"✅ Regular",pct>=75,"⚠️ At Risk",TRUE,"🔴 Problem")))
=MAP(C2:C50,D2:D50,ATTEND_STATUS)
🏆

BONUS_ELIGIBLE — Bonus check

Checks attendance + target achievement → Full / Partial / No Bonus

Arguments: target, achieved, attend_pct
=LAMBDA(t,a,ap,IF(ap<90,"❌ No (Attendance)",IF(a>=t,"✅ Full",IF(a>=t*.8,"🟡 Partial","❌ No"))))

MARGIN_BAND — Margin classifier

Sale and cost → High / Medium / Low margin

Arguments: sale, cost
=LAMBDA(s,c,LET(m,(s-c)/s*100,IFS(m>=30,"🟢 High",m>=15,"🟡 Medium",TRUE,"🔴 Low")))
=MAP(C2:C200,D2:D200,MARGIN_BAND)

ORDER_VALUE — Final invoice total

Qty × price → apply discount → add GST

Arguments: qty, price, disc_pct, gst_pct
=LAMBDA(q,p,d,g,LET(gr,q*p,(gr-gr*d/100)*(1+g/100)))

Try it Yourself

Five exercises — from simple to challenge level. Read the problem, write the LAMBDA yourself first, then check the answer.

Mark a payment as Paid or Unpaid
Easy

Problem: Column C has the payment date. If it's blank → "❌ UNPAID". If it has a date → "✅ PAID".

Answer — Name: PAY_STATUS · Argument: payment_date

2
Net Profit percentage
Easy

Problem: You have revenue, COGS, and operating expenses. Return "Net Profit: 18.5%".

Answer — Name: NET_PROFIT · Arguments: revenue, cogs, opex
=LAMBDA(r,c,o,"Net Profit: "&ROUND((r-c-o)/r*100,1)&"%")
3
Find best subject score for each student using BYROW
Medium

Problem: Students in rows, 5 subjects in columns B to F. Show the highest score each student got.

=BYROW(B2:F50, LAMBDA(row, MAX(row)))

Running total of daily collections using SCAN
Medium

Problem: Column B has daily cash collected. Create a column showing cumulative total at each day.

=SCAN(0, B2:B30, LAMBDA(total, day, total+day))

One-line salary slip summary
Advanced
